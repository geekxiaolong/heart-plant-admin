# 产品需求文档 (PRD) 优化：多元化认领逻辑 (Inclusive Onboarding)

## 1. 认领逻辑重构：从“情感契约”到“守护模式”
我们将认领流程从单一的社交驱动，改为**双轨道驱动**。将“个人爱好”包装为**“自我独处/自然治愈”**模式，使其与“亲情/爱情/友情”并列，保持视觉逻辑的一致性。

---

## 2. 优化后的认领步骤 (Updated User Flow)

### 2.1 第一步：心动选择 (Discovery)
* **交互**：用户在首页点击感兴趣的植物，进入详情页。
* **信息展示**：除了基础参数，增加“养护心情”标签（如：#治愈、#陪伴、#挑战）。

### 2.2 第二步：定义你的守护模式 (Choose Your Mode)
系统弹出模式选择页，将“个人爱好”无缝融入其中，设计为四大平行选项：

| 模式名称 | 适用场景 | 核心价值 | UI 风格 |
| :--- | :--- | :--- | :--- |
| **悦己模式 (Solo)** | **个人爱好/自我解压** | 专注植物成长，独享宁静 | **极简工业风 / 禅意青** |
| **亲情模式 (Kinship)** | 父母子女/长辈陪伴 | 跨越距离的关怀 | 暖阳橙 |
| **爱情模式 (Romance)** | 情侣/爱人 | 共同守护的浪漫 | 浪漫粉 |
| **友情模式 (Friendship)** | 好友/知己 | 共同话题与乐趣 | 活力绿 |

### 2.3 第三步：仪式感定制
* **个人认领（悦己模式）**：
    * **入场词**：系统提示“欢迎开启你的私人治愈时光”。
    * **目标设定**：设定一个养护目标（如：期待它开花、坚持打卡30天）。
    * **跳过邀请**：直接进入个人互动的花园。
* **多人认领（情感模式）**：
    * **关系绑定**：触发原有的“邀请合伙人”流程。
    * **互动锁定**：需对方接受后开启完整功能。

### 2.4 第四步：互动页的差异化表现
* **个人模式下**：
    * 互动页的“情感记录”转化为**“养护日记”**。
    * 界面更强调数据可视化（更专业的生长曲线图）。
    * 移除“对方在线”状态，改为“专注时长”统计。
* **情感模式下**：
    * 保持原有的实时协同、双人足迹、情感弹幕功能。

## 3. UI 布局优化描述 (供 Gemini 参考)

> **指令 (针对个人/悦己模式):**
> "设计一个‘悦己模式’的植物控制页面。
> 1. **风格**：极简、禅意，使用冷灰或深绿色调 (#2F4F4F)。
> 2. **重点展示**：将 IoT 传感器数据（温湿度）与植物的生长曲线图结合，展现专业感。
> 3. **记录功能**：将记录入口命名为‘养护手记’，支持拍照上传植物微距照片。
> 4. **氛围感**：背景可以有微弱的白噪音波动线条。"

---

## 4. 逻辑细节处理 (Logic Details)

* **模式转换**：允许用户在认领后（如果原本是个人模式）通过“邀请好友”升级为“协同模式”，但不可逆向降级（保护另一方的记录隐私）。
* **首页区分**：在首页认领列表中，个人认领的卡片展示“我的进度”，共同认领的卡片展示“我们的故事”。
