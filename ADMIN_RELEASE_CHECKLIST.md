# Admin 发布就绪检查清单

更新时间：2026-03-10

## 当前判断

- 构建状态：`npm run build` 通过
- 前端稳定性：本轮改动已把后台主要页面统一到同一套鉴权请求头与响应解析逻辑，明显降低了因 payload 形态不一致、路径插值错误、列表不刷新导致的假故障
- 发布建议：**可进入联调 / 预发布验证**，但正式放量前仍建议做一次带真实管理员账号的手工回归

## 本轮稳定项

- 管理员邮箱判定统一转小写，减少大小写导致的误判
- API 基础层统一：
  - 本地 `127.0.0.1` 走本地后端
  - 登录后请求头统一携带用户 token
  - 统一解析 `{ data | result | items }` 等 payload 形态
  - 统一识别 `success: false` 失败响应
- 修复多处字符串插值错误，避免请求落到字面量路径
- 植物库 / 日记 / 时间线 / 仪表盘 / 认领页统一接入更稳健的错误处理
- 新增 / 编辑植物支持直接填写图片 URL，减轻上传链路不稳定时的阻塞
- 列表页在新增、编辑、删除、精选等动作后可按路由 state 自动刷新
- 修复若干后台导航回跳错误（如 `/stream-test`、`/interaction`）

## 仍需人工验证的关键流

### 1. 登录与权限
- [ ] 使用真实管理员账号登录 `/admin/login`
- [ ] 已登录管理员可进入 `/admin`
- [ ] 非管理员账号会被 `AdminGuard` 拦截
- [ ] 失效会话会跳回登录页

### 2. 植物库
- [ ] 植物列表可正常加载
- [ ] 新增植物：直接填写图片 URL 后可保存
- [ ] 编辑植物：修改名称 / 文案 / 图片 URL 后可保存
- [ ] 删除植物行为与当前后端能力一致（成功删除，或明确返回受支持/不支持提示）
- [ ] 新增/编辑/删除后列表会自动刷新

### 3. 日记与时间线
- [ ] 日记列表可正常加载
- [ ] 日记详情可打开
- [ ] 精选切换行为与当前后端能力一致
- [ ] 删除日记行为与当前后端能力一致
- [ ] 返回日记列表后状态能自动刷新
- [ ] 植物时间线可按 plantId 正常切换与展示

### 4. 仪表盘与认领视图
- [ ] Dashboard 统计卡片可正常显示，不因 payload 结构变化报错
- [ ] 已认领植物列表可正常显示
- [ ] 认领管理页可正常显示

### 5. 本地联调
- [ ] 在 `127.0.0.1` 环境确认 API 已走本地 `8000`
- [ ] 本地后端鉴权头能正确读取 `Authorization` / `X-User-JWT`

## 残余低优先级风险

- `ADMIN_AUTH_AUDIT.md` 对后端能力现状的描述可能已落后于代码，需要和 `heart-plant-api` 实际实现再对一次
- `backendCapabilities.ts` 目前是静态开关；若后端能力按环境不同，后续更适合改为运行时探测或环境配置
- 构建产物仍有大 chunk 警告，暂不阻塞发布，但值得后续做拆包优化
- 目前管理员判定仍包含前端硬编码邮箱，长期建议收敛到统一的后端 / claims 角色判定

## 建议发布门槛

满足以下条件即可认为 admin 侧可发布：

- [ ] 真实管理员账号登录成功
- [ ] 植物新增 / 编辑 / 删除至少各跑通一次
- [ ] 日记详情 / 精选 / 删除至少验证一次
- [ ] Dashboard、认领页、时间线无白屏无控制台致命错误
- [ ] 与 `heart-plant-api` 对齐当前支持接口列表
